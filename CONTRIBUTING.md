# Contributing to Ergentum

First of all — thank you for being here. Every contribution matters, no matter how small.

Ergentum is built for everyone. That means contributions are welcome from everyone too — developers, writers, translators, designers, and domain experts who want to build bots.

---

## Ways to contribute

You don't need to be a developer to contribute. Here are all the ways you can help:

### 🌍 Translate the documentation
The Wiki is currently in English. We need translations into Spanish, Portuguese, French, German, and Japanese. If you're a native speaker, this is one of the most impactful contributions you can make.

See Issue [#6](https://github.com/Ergentum/ergentum-protocol/issues/6) for details.

### 📝 Improve the documentation
Found something confusing? A step that doesn't work? A question the FAQ doesn't answer? Open an issue or submit a pull request with your improvement.

Good documentation is never finished. Every person who reads it and finds something unclear can make it better for the next reader.

### 🐛 Report a bug
If something doesn't work as expected, open an issue. Include:
- What you were trying to do
- What happened instead
- Your operating system and cardano-cli version

### 💡 Suggest a feature
Have an idea? Open an issue with the label `enhancement`. Describe the problem it solves and how you imagine it working.

### 🖥️ Run a node
Running a node is a direct contribution to network decentralisation. Every node makes Ergentum more resilient.

See [How to run a node](https://github.com/Ergentum/ergentum-protocol/wiki/How-to-run-a-node) to get started.

### 🤖 Build a bot
Create a specialised bot in your area of expertise. Every bot adds value to the network and earns you ERGON automatically.

See [How to create a bot](https://github.com/Ergentum/ergentum-protocol/wiki/How-to-create-a-bot) to get started.

### 💻 Contribute code
Check the open issues — especially those marked [`good-first-issue`](https://github.com/Ergentum/ergentum-protocol/issues?q=is%3Aissue+is%3Aopen+label%3Agood-first-issue). These are specifically chosen for contributors who are new to the project.

## How to submit a pull request

1. **Fork the repository**
   Click the Fork button at the top right of this page.

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/ergentum-protocol
   cd ergentum-protocol
   ```

3. **Create a branch**
   ```bash
   git checkout -b your-branch-name
   # Example: git checkout -b fix/node-onboarding-typo
   # Example: git checkout -b docs/spanish-translation
   ```

4. **Make your changes**
   Keep changes focused. One pull request = one thing.

5. **Test your changes**
   If you're modifying the onboarding script, test it end-to-end. If you're editing documentation, read it out loud — if it sounds awkward, rewrite it.

6. **Commit with a clear message**
   ```bash
   git commit -m "fix: correct typo in node onboarding step 3"
   git commit -m "docs: add Spanish translation of What is Ergentum"
   git commit -m "feat: add automatic dependency check to onboard script"
   ```

7. **Push and open a pull request**
   ```bash
   git push origin your-branch-name
   ```
   Then go to GitHub and open a pull request against the `main` branch.

---

## Commit message format

We use a simple format:

| Prefix | Use for |
|--------|---------|
| `feat:` | New feature or functionality |
| `fix:` | Bug fix |
| `docs:` | Documentation changes |
| `refactor:` | Code restructuring without behaviour change |
| `security:` | Security improvements |
| `chore:` | Maintenance tasks |

## Documentation contributions

The Wiki lives at `https://github.com/Ergentum/ergentum-protocol/wiki`.

To contribute to the Wiki:
1. Clone the wiki repository separately:
   ```bash
   git clone https://github.com/Ergentum/ergentum-protocol.wiki.git
   ```
2. Edit the markdown files
3. Push your changes

For translations, create new files with the language code suffix:
```
What-is-Ergentum-ES.md  ← Spanish
What-is-Ergentum-PT.md  ← Portuguese
What-is-Ergentum-FR.md  ← French
```

## Code of conduct

Ergentum is built on the principle that the best outcomes come from voluntary cooperation between people with diverse backgrounds.

We expect all contributors to:

- **Be respectful** — disagree with ideas, not people
- **Be honest** — if something doesn't work, say so
- **Be patient** — not everyone has the same experience level
- **Be constructive** — criticism is welcome when it comes with a suggestion

Contributions that violate these principles will not be accepted.
